入驻简书-2017第一篇

翻翻自己写的博客，最近一次跨年整理思绪是在2014年年初，至今已有3年光景，在这期间其实是有几次想提笔记录的，但是，总觉得技术及事业上摇摇晃晃，并未定性，因此也搁置了。

直至2016年10月来到樊登读书会企业版的团队中，到现在约3个月有余，才真正觉得是时候安定下来了，因此打算更一下博，顺便入驻一下简书。

2014-2016年的一些总结

14年在朋友劝说下，开始进入创业团队的管理层，看见了很多之前专注技术下的很多无法触碰的东西，也给了我很多启示，学到了很多。自己Team的方方面面需要考虑到，在管理是，每一位同仁，要怎么样的去对待？既要不能让大家失望，又要不让公司失望，比较难。当时我的处理方式是，讨好上司，讨好部下。

15年，创业公司失败，重新去到高级开发，此时因为技术全面，Title被给到架构师，很开心，因为一直往这个方向努力。值得一提的是，15年所在团队，是目前我从业十来年所处的最开心的一个团队，这里有非常开明的技术领导人，有同龄的开发，有相当年轻的团队成员。加上本人年轻的心，实在相得益彰啊。

现如今，来到樊登读书会企业版，一手搭建的技术团队，短短3个月，做了相当多的工作，2个APP都要上线了，后端的几个管理系统也是相当的好。Team实在是给力。

总的来说，是摇摇晃晃的几年，又沉淀了很多技术的管理的东西。

2017年的一些想法

翻开14年总结的博文，基本上完成了一些，放弃了一些，还要继续一些。

时值2017，手上的事业摆到第一位，经历了那么多技术，都是为业务和产品服务的，只有到某一阶段，技术才会真正达到非常重要的高度。这既是所谓的短期技术高估，长期技术低估的由来。

本年最重要的任务是，持续的学习，持续的整理，持续的深入。

持续学习

作为码农，从来不会因为走向管理职位，而放弃或者看轻技术的，因为从心里深深的知道，技术立本。

简单的几个例子，一项事业，包含很多产品，业务。纵向上看有很多阶段。两个维度上看，所使用的技术需要很多面的，主业务 Java ，Javascript , Objective-C。业务周边需要 Python 等动态的语言。所以学习，从不应该被停止。

持续整理

管理上，技术上，个人修养上，需要不停的去反复思考，把看到的，碰到的默默记录下来，看看是否有更好的方式，每日三省。

自我的知识体系目前需要好好规划了，后10年，可能就靠这些了。。。

持续深入

以前年轻时喜欢折腾各种玩意儿，藏书三百卷有余，大部分翻翻，记下目录，用时还是觉得不够.

现在想来，广度上，实在不是这个年纪的人做的事情，那么，从深度上看，似乎更合适点。

列出几个完成的点，给自己点安慰；列出几个放弃的，让自己清醒；再列出几个未完成的和需要完成的，给自己点动力。

完成的

  • MySQL 使用上，运维上略有心得。
  • 软件管理，Scrum 大部分概念清楚，现实工作中，提炼了一些点，非常受用。
  • Java 部分，3年撸了2次多 core Java 和 Thinking in Java，略满意
  • Redis 撸过1遍，须继续深入
  • MongoDB 撸过1遍，继续深入

放弃的

  • Rails

未完成的

  • Java 并发及多线程
  • 架构的总结，DDD 的更多实践
  • 团队管理

入驻简书

简书是个非常棒的平台，从UE/X上看，非常 Web 2.0，用着很舒服，打算将自己的博客上的技术文章，做个同步更新。主要目的是记录+梳理。

展望

事业上希望更进一步，发挥一下在处女座领导那里学习来的工作习惯。

技术上，是时候继续深入了。

生活上，希望家人幸福安康。
